Fluent Interface

Recently, I’ve played with my own framework I found a new term, called “Fluent Interface”. There is great article on blog of Martin Fowler about this term. The idea of “Fluent Interface” is instead of returning void in setter methods of an object, returning an object to promote object chaining. Example: class Query { private String props; private String table; private String condition; public Query select(String props) { this.props = props; return this; } public Query from(String table) { this.table = table; return this; } public Query where(String condition) { this.condition = condition; return this; } public String buildQuery() { return String.format("SELECT %s FROM %s WHERE %s", props, table, condition); } } Instead of: Query q = new Query(); q.select("username"); q.from("user"); q.where("username = 'duydo'"); String sql = q.buildQuery(); We can write: Query q = new Query(); String sql = q.select("username").from("user").where("username = 'duydo'").buildQuery(); This will make the code more readable.
