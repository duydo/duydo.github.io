JPA Repository: Make your data access layer easy

This is a small lib I’ve created to implement repository programming model with JPA 2 and http://en.wikipedia.org/wiki/Specification_pattern during learning and working with JPA, it provides a simple and easy way to build the data access layer. The source code is hosted at https://github.com/duydo/jpa-repository

If you have any suggestion or feedback, don’t hesitate to drop me a line :-)

Build & Install

It is very easy to build from source code with maven:

git clone https://github.com/duydo/jpa-repository.git
cd jpa-repository
mvn install

If you use maven with the project, just adding dependency to your pom.xml

<dependency>
    <groupId>com.duydo</groupId>
    <artifactId>jpa-repository</artifactId>
    <version>0.0.1-SNAPSHOT</version>
</dependency>

Usage

Create an entity, User for example:

    @Entity
    public class User extends EntityObject {
        private String name;
        private Integer age;
        private String email;

        public User() {}
        //getter/setter here
    }

Create an instance of Repository:

EntityManager em = ...;
Repository repo = new JPARepository(em);

Basic Queries

//find all users
List users = repo.findAll(User.class)

//find user by id
User user = repo.findById(User.class, 1L);

//create
User user = new User();
repo.save(user);

//update
User user = repo.findById(User.class, 1L);
user.setName("Duy Do");
repo.update(user);

// delete
repo.remove(user);

Query with Specification

// Find unique user has email abc@example.com
Specification hasEmail = Specifications.equal("email", "abc@example.com");
User user = repo.findBySpecification(User.class, hasEmail).asSingle();


// Find all users has name “Duy”
Specification hasName = Specifications.equal("name", "Duy");
List users = repo.findBySpecification(User.class, hasName).asList();

//Find all users which name is not “Duy”
Specification notName = Specifications.equal("name", "Duy").not();
List users = repo.findBySpecification(User.class, notName).asList();

// We can combine specifications with AND, OR
//Find users has name "Duy" and age is "28"
Specification hasName = Specifications.equal("name", "Duy");
Specification hasAge28 = Specifications.equal("age", 28);
List users = repo.findBySpecification(User.class, hasName.and(hasAge28)).asList();

Sorting

Sorting by using method sortXXX() of SepecificationResult

// Find all users has name “Duy” and sort ascending by name

Specification hasName = Specifications.equal("name", "Duy");
List users = repo.findBySpecification(User.class, hasName).sortAscending("name").asList();

Paging

Paging by using method skip(int count) and get(int count) of SepecificationResult

//Find 10 users has name “Duy”
Specification hasName = Specifications.equal("name", "Duy");
List users = repo.findBySpecification(User.class, hasName).get(10).asList();

// Find all users has name “Duy” but skip 10 first users
Specification hasName = Specifications.equal("name", "Duy");
List users = repo.findBySpecification(User.class, hasName).skip(10).asList();

//Find 20 users has name “Duy” but skip 10 first users
Specification hasName = Specifications.equal("name", "Duy");
List users = repo.findBySpecification(User.class, hasName).skip(10).get(20).asList();
